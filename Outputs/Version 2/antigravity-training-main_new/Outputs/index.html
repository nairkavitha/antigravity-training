Module 01 · Foundations

What Is Agentic AI, and Why Does It Matter?

Before you touch the tool, let's build the mental model. This module answers three questions: what is an AI agent, how does it think, and where do you fit in?

🤖
Chatbots vs Agents: What's the Difference?
They both use AI, but they work very differently
🏗️
The Expert Contractor Metaphor Think of a traditional chatbot like a brilliant colleague you can ask questions, they'll answer anything, but they need you to do the work. An AI agent is more like hiring an expert contractor. You tell them what you want built, hand them the keys, and they figure out the steps. You still review the work and approve final decisions, but the execution is theirs.
Dimension 💬 Chatbot (e.g. ChatGPT) 🤖 AI Agent (Antigravity 2.0)
How you interact You ask questions, it answers You set a goal, it plans and executes
Can it take action? No: output is text only Yes: reads files, writes files, runs code
Memory across steps Limited to the conversation Uses context files to persist state
Can it use tools? Sometimes (via plugins) Yes: built-in tools + MCP connections
Who decides the steps? You (you tell it each step) The agent (it plans; you approve key decisions)
Best for Quick Q&A, drafting, brainstorming Multi-step tasks, file work, automation
🔄
How an Agent Thinks: Plan → Act → Verify
Every agent task follows this loop
🧠
Plan
The agent reads your goal and context files, then decides what steps to take and in what order.
Agent → you review
→
⚡
Act
The agent executes: reads files, runs code, writes outputs. It pauses at checkpoints for your approval.
HITL checkpoint
→
✅
Verify
You review the output. If it's right, you accept. If not, you correct and the loop continues.
You decide
⚠️
HITL: Human in the Loop HITL isn't optional, it's the design. The agent is powerful, but it can make mistakes, misread context, or hallucinate. Your role is to stay in the loop: review plans, approve key actions, and verify outputs. You are the quality control layer.
🔗
How the Loop Becomes Visible To run the Plan → Act → Verify loop, the agent doesn't guess or read your mind. It reads the briefing files in your workspace, and it shows you its thinking through artifacts:
  • Plan: The agent reads your workspace files (README, context) and writes an Implementation Plan artifact for your sign-off.
  • Act: It executes and ticks off a live Task Plan artifact, asking permission before sensitive actions.
  • Verify: It hands you a Walkthrough artifact plus the output files. You review, comment, approve.

What you must know before using any agent on real data
🔏
Never use real employee data
During training, use only the sandbox data in HR_Mission_Control/. Never paste real employee names, IDs, salaries, or personal data into any agent prompt.
🔴 Critical
🧾
What counts as PII
Personally Identifiable Information includes: full name, employee ID, email, phone number, date of birth, location, salary, performance rating, medical info. When in doubt, leave it out.
📋 Know This
✅

The HR_Mission_Control/ sandbox contains synthetic, fictional data, no real people. You can experiment freely without risk.
🟢 Safe to Use
🔍

Agents can hallucinate, inventing facts that sound plausible but are wrong. Any output that informs a real decision must be independently verified before use.

Knowledge Check
Tap a question to reveal the answer
What is the key difference between a chatbot and an AI agent? ⌄
A chatbot responds to your questions with text, you do the work. An AI agent takes action on your behalf: it reads files, executes code, writes outputs, and plans multi-step workflows. The key distinction is agency: the ability to act, not just answer.
What does HITL stand for, and why is it non-negotiable? ⌄
Human in the Loop. Because agents can make mistakes: misreading context, skipping steps, or hallucinating data. The human review layer is what keeps outputs safe and accurate. In HR work especially, decisions affect people's careers and livelihoods, so verification is mandatory.
Why does the agent write an implementation_plan.md before it starts writing code or editing files? ⌄
Because in the Plan phase, the agent must outline its intended path and get your explicit approval before execution. This prevents the agent from making unauthorized changes to your files or running incorrect code.
If the agent has direct file access, why does it still ask for your permission before reading or writing a file? ⌄
This is the security gate. Antigravity 2.0 cannot access files or run commands silently. You must inspect the target path and click Approve to ensure data governance is maintained and sensitive information stays within the sandbox.

Why must you create a Project before giving the agent any task?⌄
Because the Project defines what the agent is allowed to see. Until you add the HR_Mission_Control folder to a Project, the agent has no access to your data files and every exercise prompt will fail. The Project's security settings are also where your review-before-execute protection lives.
Where do you find the agent's Implementation Plan, and what do you do with it?⌄
In the Auxiliary Pane (toggle, top right). It is an artifact, not a file in your folder. Read it, comment on anything that looks wrong, and click Proceed only when you're satisfied. Signing off without reading it defeats the entire HITL design.
The agent asks permission to read a file outside your sandbox. What do you do?⌄
Reject it, and check why it tried. The Project boundary should prevent this by design, but the permission prompt is your second line of defence. Read every path before approving: inside HR_Mission_Control, approve; anything else, reject and re-brief the agent.
Module 03 · Core Skills

Goals, Tools & Subagents

Three skills that separate a power user from a frustrated one. Write goals that land first time, understand how agents use tools, and run your first multi-agent workflow.

🎯
Goals vs. Orders: The Most Important Shift
How you write your prompt determines everything
🧠
The core insight Most people default to orders: "First do this, then that." But agents are designed to figure out the steps themselves. Describe the outcome you want, not the route to get there. Goals unlock the agent's full capability. Orders constrain it unnecessarily.
📋
Step-by-step instruction
"Step 1: Open data/employees.csv. Step 2: Count the rows. Step 3: Group by department. Step 4: Write result to file."
⚠️ Rigid, brittle, micromanages the agent.
🎯
Outcome-focused goal
"Read data/employees.csv and create a team composition summary by department and grade. Save as exercises/team_summary.md."
✅ Clear, flexible, defines what and where, not how.
📝
The Anatomy of a Good Goal
1. What to do
The outcome you want. What does "done" look like?
2. What to use
Which files or data sources. Be specific about filenames.
3. Where to save it
The exact output path and filename. Don't leave this ambiguous.
4. Any constraints
Format requirements or governance rules. "Don't include salary data" is a valid constraint.

Why is an explicit file path better than "save it somewhere"?⌄
Ambiguous save instructions force the agent to make an unapproved decision: wrong folder, random name, or overwriting something important. Explicit paths are a governance mechanism. They make actions predictable and auditable.
What should you do if the output doesn't match what you expected?⌄
Don't delete and restart. Read the output carefully, the agent usually gets 80% right and needs a targeted correction. Follow up specifically: "The summary is missing the grade breakdown, please add that section." This is faster than re-running from scratch.

Understanding MCP
The protocol that connects agents to your ecosystem
🧩
MCP in plain language Model Context Protocol is a standard that lets AI agents connect to external services consistently and safely. Think of it like a universal adapter: instead of custom code for each integration, MCP provides one standard interface. Each connection is called an MCP server.
MCP ServerWhat It UnlocksHR Use Case
📁 File SystemRead/write local files and foldersGenerating reports, processing employee data
💬 SlackRead channels, send messagesDrafting announcements, searching HR queries
📄 NotionRead/write pages and databasesUpdating policy docs, creating onboarding wikis
📅 Google CalendarRead events, create meetingsScheduling reviews, setting up interview blocks
📊 Google DriveRead/write Sheets and DocsUpdating org charts, processing survey responses
⚠️
MCP requires approval Every MCP connection must be configured by your IT/platform team. Do not attempt to connect to production systems (Workday, SAP, HRMS) without explicit approval. During training, use only file system tools and sandbox data.

Knowledge Check
Tap a question to reveal the answer
When should you use subagents vs a single agent?⌄
Use subagents when: (1) the task has clearly distinct, parallel workstreams; (2) each workstream needs different tools or data sources; (3) the combined output is too large for one agent's context window. For most daily tasks, a single agent is simpler and faster. Subagents shine in pipelines and multi-source processing.
What if one subagent fails or produces bad output?⌄
Always verify individual subagent outputs before the merge. If the People Analyst section is wrong, fix it before the orchestrator merges, bad input creates a bad report even if the merge itself is perfect. The orchestrator will typically report an error, but don't rely on it catching everything.

Tap a question to reveal the answer
The agent's first version looks perfect. What do you do before sharing it with anyone?⌄
Audit it against the source. Pick at least one record and trace it from the data file to what the tool displays. Beautiful and correct are different properties; a quick look only proves the first one.
The output is 80% right. Do you restart or refine?⌄
Refine. Point at the exact gap: "The summary is missing the grade breakdown, add that section, change nothing else." Targeted feedback preserves everything that's already right; restarting throws it away and often reintroduces old mistakes.
When is the right moment to add constraints like "no salary data"?⌄
When you write the goal, not after the build. Outputs travel: they get screenshared and forwarded. A tool that never contained sensitive data can't leak it. Constraints in the goal are governance by design; cleanups afterwards are governance by luck.

When Things Go Wrong: Your Recovery Toolkit

Every agent user hits the same four or five failure patterns. This module gives you the exact steps to diagnose and fix each one, without panic, without starting over.

🛡️
The single most important recovery skill Before trying anything below, read the Thinking Stream. The agent almost always tells you exactly what went wrong, in plain language. Most problems are solved by reading the error message carefully, not by guessing.
🔧
Common Failure Patterns
Click to expand diagnosis + fix
🌀
The agent is going in circles: it keeps repeating the same steps
Infinite Loop ⌄

Why this happens: The agent's goal is ambiguous or the success condition is unclear, so it keeps trying variations without knowing when to stop.

  1. Press Esc to interrupt immediately
  2. Read the last 5–10 lines of the Thinking Stream: what is it trying to do?
  3. Rewrite your goal with a clearer success condition. Add: "Stop when you have produced a file at [path]. Do not iterate further."
  4. If the agent has created partial output, check it, you may be able to correct it rather than re-run from scratch
recovery prompt
Stop. Review what you have done so far and tell me: 1. What output have you already produced? 2. What is stopping you from completing the task? 3. What is the single next step needed to finish? Do not take any further action until I have reviewed your answer.
🚫
Permission denied: the agent can't read or write a file
Permission Error ⌄

Why this happens: The agent is trying to access a path it doesn't have permission to, or the path doesn't exist yet.

  1. Check the error message, it will name the exact file or folder it can't access
  2. Open Settings → your Project and confirm the HR_Mission_Control folder is actually added to the Project
  3. If the output folder doesn't exist yet, tell the agent: "Create the exercises/ folder first, then save there" and approve the permission prompt
  4. If it's a file read error, confirm the filename and path are exactly right (case-sensitive on macOS/Linux)
  5. If a permission prompt was rejected by mistake, just re-run the request and approve it this time
diagnostic prompt
List all files and folders in the current directory. Then tell me your current working directory path. Do not take any other action until you have done this.
🧠
The output looks wrong: data seems invented or incorrect
Hallucination ⌄

Why this happens: The agent "filled gaps" in the data with plausible-sounding but invented information. This is more common when the source data is incomplete or ambiguous.

  1. Open the source file (e.g. data/employees.csv) and cross-check against the output
  2. Look for: numbers that don't add up, names that don't appear in the source, policies that don't exist in the policies/ folder
  3. Do not use hallucinated output for any real decision
  4. Prompt the agent to cite its sources, see prompt below
  5. If the hallucination is widespread, restart with a more constrained goal that explicitly says "only use data from the provided file, do not invent or estimate any values"
verification prompt
For every data point in the output you just produced: - Tell me which source file it came from - Tell me the exact row or section it was taken from If any value was estimated, calculated, or assumed rather than directly read from a file, flag it clearly with [ESTIMATED].
🗺️
I'm lost: I don't know where the agent is up to
Disorientation ⌄

Why this happens: Long or complex tasks can drift, especially if you've been iterating for a while. The agent's context may be getting large, and it's easy to lose track of what has actually been completed.

  1. Ask the agent for a status summary (prompt below) before doing anything else
  2. Check the exercises/ folder: what files actually exist?
  3. Read the most recent output file: does it look complete and correct?
  4. If the conversation is very long, start a fresh conversation in the same Project (click + next to the Project name) with a more focused goal that picks up from where you are
status check prompt
Pause and give me a progress report. Tell me: 1. What was the original goal? 2. What have you completed so far? (list each output file created) 3. What is still outstanding? 4. Are there any blockers or errors to be aware of? Do not take any further action until I have reviewed this.
⚡
The agent stopped halfway: it seems stuck or timed out
Incomplete Run ⌄

Why this happens: Long tasks can hit context limits, network timeouts, or simply stall waiting for a response the agent isn't sure how to give.

  1. Check the exercises/ folder, the agent may have saved partial output you can build from
  2. Check the Auxiliary Pane, the Task Plan artifact shows exactly which steps completed
  3. Do not abandon the conversation yet, use the continuation prompt below first
  4. If it doesn't recover, start a new conversation in the same Project with the continuation goal, using existing output as your starting point
continuation prompt
You were in the middle of a task and seem to have stopped. Read all files currently in the exercises/ folder and tell me: 1. What has already been completed? 2. What is the next step to finish the original task? Then continue from where you left off, do not re-do completed work.
